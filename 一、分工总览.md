### 二、详细任务清单

#### 🔹 成员1—— 院系A + 课程共享 + XML验证

**院系A开发任务**

- 设计SQL Server数据库表（账户、学生、课程、选课），参考PDF表3-2~3-5
- 生成50名学生、10门课程、每人5门选课（可使用成员4提供的统一数据脚本）
- 实现A的图形界面（Java Swing/JavaFX）：
  - 学生/管理员登录
  - 查看本院课程、选课/退课（本系内）
  - 查看共享课程（从集成服务器拉取）
- 实现A的本地XML接口：
  - 读取数据库生成本地格式XML（使用DOM4J）
  - 解析收到的XML并写入数据库
- 实现A与集成服务器的HTTP通信（发送请求、接收响应）

**集成服务器任务**

- **XML验证模块**：编写XSD验证器（基于Xerces），验证各院系发来的XML是否符合预定Schema
- **课程共享流程**（参照PDF图3-13）：
  - 接收院系A的共享请求 → 向院系B、C请求课程XML → 验证格式 → 转换为统一格式 → 再转换为A格式 → 返回A
- 负责最终代码合并和集成启动脚本编写

**交付物**

- 院系A完整源代码及可运行程序
- XML验证模块代码
- 课程共享流程实现代码

---

#### 🔹 成员2—— 院系B + 跨系选课 + XML转换

**院系B开发任务**

- 设计Oracle数据库表（参考PDF表3-6~3-9）
- 生成50名学生、10门课程、每人5门选课
- 实现B的GUI（字段与A略有差异，需适配）
- 实现B的本地XML接口和与集成服务器的通信

**集成服务器任务**

- **XML转换模块**：编写XSLT转换器（Java Transformer API）
  - 实现各院系本地格式 ↔ 统一格式的XSL文件（如formatClass.xsl、classToA.xsl等）
  - 提供完整的XSLT文件列表供成员1和成员3调用
- **跨系选课流程**（参照PDF图3-14）：
  - 接收跨系选课请求 → 转换学生/课程XML为统一格式 → 转发至目标院系 → 返回结果

**交付物**

- 院系B完整源代码及可运行程序
- 所有XSLT转换文件
- 跨系选课流程实现代码

---

#### 🔹 成员3—— 院系C + 统计 + 退课 + 通信框架

**院系C开发任务**

- 设计MySQL数据库表（参考PDF表3-10~3-11及后续）
- 生成50名学生、10门课程、每人5门选课
- 实现C的GUI和XML接口

**集成服务器任务**

- **通信框架**：搭建简易HTTP Server（可使用`com.sun.net.httpserver`或Spring Boot简版）
  - 定义路由：`/share`（课程共享）、`/crossSelect`（跨系选课）、`/crossDrop`（跨系退课）、`/statistics`（统计）
  - 实现请求解析和响应封装（XML格式）
- **统计功能**：
  - 向A、B、C分别请求学生、课程、选课XML
  - 转换为统一格式后统计：总学生数、总课程数、总选课人次
  - 返回统计结果（可显示在集成服务器控制台或简单Web页面）
- **跨系退课流程**（新增，任务要求）：
  - 接收学生退课请求 → 解析课程所属院系 → 转发至该院系删除选课 → 返回结果

**交付物**

- 院系C完整源代码及可运行程序
- 通信框架代码
- 统计功能和跨系退课流程实现代码
